Introduction to HTML

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It is the most basic building block of the Web and defines the meaning and structure of web content.

Unlike programming languages that use scripts to perform functions, HTML uses tags to identify content and give it structure. Web browsers read HTML documents and render them into visible or audible web pages.

History of HTML

HTML was created by Tim Berners-Lee in 1991 while working at CERN. The first version had 18 elements, which were mostly based on SGML (Standard Generalized Markup Language). Since then, HTML has evolved through multiple versions:

  • HTML 2.0 (1995): The first standardized version
  • HTML 3.2 (1997): Added tables, applets, and text flow around images
  • HTML 4.01 (1999): Introduced stylesheets, scripting, and accessibility features
  • XHTML (2000): Reformulation of HTML as an XML application
  • HTML5 (2014): Current version with new semantic elements, multimedia support, and APIs

Why Use HTML?

HTML serves several crucial purposes in web development:

  • Structure Content: HTML provides the basic structure of sites, which is enhanced and modified by other technologies like CSS and JavaScript.
  • Universal Standard: As a web standard maintained by the W3C (World Wide Web Consortium), HTML works across all browsers and devices.
  • Accessibility: Proper HTML structure makes web content accessible to people with disabilities and compatible with assistive technologies.
  • SEO Friendly: Search engines rely on HTML structure to understand and index web content effectively.
  • Integration: HTML works seamlessly with other web technologies like CSS for styling and JavaScript for interactivity.

How to Create HTML Documents

Creating HTML documents is straightforward. You only need a basic text editor and a web browser:

  1. Open a text editor (Notepad, TextEdit, VS Code, etc.)
  2. Create a new file and save it with a .html extension
  3. Write your HTML code
  4. Save the file and open it in a web browser
<!-- Basic HTML Document Structure --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Document Title</title> </head> <body> <!-- Content goes here --> </body> </html>
Note: The <!DOCTYPE html> declaration must be the very first thing in your HTML document, before the <html> tag. This declaration is not an HTML tag; it is an instruction to the web browser about what version of HTML the page is written in.

HTML Elements

HTML elements are the building blocks of web pages. Each element represents some kind of structure or semantics and generally consists of a start tag, content, and an end tag.

Anatomy of an HTML Element

<!-- Structure: <tagname attribute="value">Content</tagname> --> <a href="https://www.example.com" target="_blank">Visit Example</a>
  • Start tag: <a href="https://www.example.com" target="_blank">
  • Attributes: href and target provide additional information
  • Content: "Visit Example" - what users see
  • End tag: </a> - closes the element

Categories of HTML Elements

1. Block-level Elements

Block-level elements start on a new line and take up the full width available. Examples include:

<div>Division or section</div> <p>Paragraph</p> <h1>Heading 1</h1> <ul>Unordered list</ul> <li>List item</li> <section>Section</section>

2. Inline Elements

Inline elements do not start on a new line and only take up as much width as necessary. Examples include:

<span>Inline container</span> <a>Anchor (link)</a> <img>Image</img> <strong>Strong importance</strong> <em>Emphasis</em>

3. Void Elements

Void elements (or empty elements) have no content and no closing tag. Examples include:

<br> <!-- Line break --> <hr> <!-- Horizontal rule --> <img src="image.jpg" alt="Description"> <input type="text">

Common HTML Elements Reference

Element Description Example
<html> Root element of an HTML page <html>...</html>
<head> Container for metadata <head><title>Page</title></head>
<body> Document's content <body>Content here</body>
<h1> to <h6> HTML headings <h1>Main Heading</h1>
<p> Paragraph <p>This is a paragraph</p>
<a> Hyperlink <a href="url">Link text</a>
<img> Image <img src="img.jpg" alt="Description">
<ul>, <ol>, <li> Unordered list, Ordered list, List item <ul><li>Item</li></ul>

<span> Inline container <span>Inline content</span>
Best Practice: Always use the appropriate HTML element for its intended purpose. For example, use button elements for buttons rather than styled divs, as native buttons have built-in accessibility features and keyboard navigation.

Semantic HTML

Semantic HTML refers to using HTML elements that clearly describe their meaning in a human- and machine-readable way. Elements like <header>, <footer>, and <article> are semantic because they clearly define their content.

Why Semantic HTML Matters

  • Accessibility: Screen readers and other assistive technologies rely on semantic markup to properly interpret and navigate content.
  • SEO: Search engines give higher priority to content within semantic elements that help them understand the page structure.
  • Maintainability: Semantic markup makes code easier to read and maintain for developers.
  • Future-Proofing: As browsers and devices evolve, semantic elements are more likely to be supported correctly.

Semantic vs Non-Semantic Elements

Non-Semantic HTML (Avoid)

<div id="header"> <div class="nav"> <div class="menu">Home</div> </div> </div> <div id="main"> <div class="article">Content</div> </div> <div id="footer">Footer</div>

Semantic HTML (Recommended)

<header> <nav> <ul> <li><a href="#">Home</a></li> </ul> </nav> </header> <main> <article>Content</article> </main> <footer>Footer</footer>

Key Semantic Elements in HTML5

Element Description Usage
<header> Represents introductory content Typically contains headings, logos, navigation
<nav> Defines navigation links Main menu, table of contents, pagination
<main> Specifies the main content Should be unique to the document
<article> Self-contained composition Blog post, news story, forum post
<section> Thematic grouping of content Chapters, tabbed content, numbered sections
<aside> Content indirectly related to main content Sidebars, pull quotes, advertising
<footer> Represents a footer for its section Contact info, copyright, related documents
<figure> & <figcaption> Represents self-contained flow content Images, diagrams, photos with captions
<time> Represents a specific period in time Dates, times, durations
<mark> Represents highlighted text Search results, highlighted content

Complete Semantic HTML Example

<header> <h1>Website Title</h1> <nav> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> </ul> </nav> </header> <main> <article> <h2>Article Heading</h2> <p>Published on <time datetime="2023-01-01">January 1, 2023</time></p> <p>Article content...</p> <section> <h3>Comments</h3> <article class="comment"> <p>Great article!</p> </article> </section> </article> <aside> <h3>Related Posts</h3> <ul> <li><a href="#">Another Post</a></li> </ul> </aside> </main> <footer> <p>© 2023 Company Name</p> </footer>

HTML Forms

HTML forms are used to collect user input. A form contains input elements like text fields, checkboxes, radio buttons, submit buttons, and more. Forms are essential for interactive websites, enabling user registration, contact forms, search functionality, and more.

Basic Form Structure

<form action="/submit-form" method="post"> <!-- Form elements go here --> </form>
  • action: Specifies where to send the form data when submitted
  • method: Specifies the HTTP method to use (get or post)

Common Form Elements

Form Example

<form action="/submit" method="post"> <div> <label for="name">Name:</label> <input type="text" id="name" name="name" required> </div> <div> <label for="email">Email:</label> <input type="email" id="email" name="email" required> </div> <div> <label for="password">Password:</label> <input type="password" id="password" name="password" required> </div> <div> <label for="message">Message:</label> <textarea id="message" name="message" rows="4"></textarea> </div> <div> <label><input type="checkbox" name="subscribe"> Subscribe to newsletter</label> </div> <button type="submit">Submit</button> </form>

HTML5 Form Enhancements

HTML5 introduced several new input types and attributes that improve forms:

Input Type Description Example
email For email addresses with validation <input type="email">
url For web addresses with validation <input type="url">
tel For telephone numbers <input type="tel">
number For numerical input <input type="number" min="1" max="10">
date For date selection <input type="date">
color For color selection <input type="color">
range For slider controls <input type="range" min="0" max="100">

Form Validation

HTML5 provides built-in form validation attributes:

<input type="text" required> <!-- Field must be filled --> <input type="email" required> <!-- Must be valid email --> <input type="number" min="1" max="100"> <!-- Number range --> <input type="text" pattern="[A-Za-z]{3}"> <!-- Regex pattern --> <input type="text" maxlength="10"> <!-- Character limit -->
Best Practice: Always use the <label> element with the for attribute matching the input's id. This improves accessibility and usability by making the label clickable, which focuses the associated input field.

HTML Accessibility

Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. HTML provides many features to make web content more accessible.

Why Accessibility Matters

  • Inclusion: Approximately 15% of the world's population lives with some form of disability.
  • Legal Requirements: Many countries have laws requiring digital accessibility.
  • Better UX: Accessible websites often provide a better experience for all users.
  • SEO Benefits: Many accessibility practices align with SEO best practices.

Key Accessibility Principles

1. Perceivable

Information and user interface components must be presentable to users in ways they can perceive.

  • Provide text alternatives for non-text content
  • Provide captions and other alternatives for multimedia
  • Create content that can be presented in different ways
  • Make it easier for users to see and hear content

2. Operable

User interface components and navigation must be operable.

  • Make all functionality available from a keyboard
  • Give users enough time to read and use content
  • Do not use content that causes seizures
  • Help users navigate and find content

3. Understandable

Information and the operation of the user interface must be understandable.

  • Make text readable and understandable
  • Make content appear and operate in predictable ways
  • Help users avoid and correct mistakes

4. Robust

Content must be robust enough to work with current and future user tools.

  • Maximize compatibility with current and future user tools

HTML Accessibility Features

Semantic HTML

Using semantic elements is one of the most important aspects of HTML accessibility:

<!-- Instead of non-semantic divs --> <div class="header">...</div> <div class="nav">...</div> <!-- Use semantic elements --> <header>...</header> <nav>...</nav>

Text Alternatives

Always provide text alternatives for non-text content:

<!-- Good: Image with alt text --> <img src="logo.png" alt="Company Logo"> <!-- Bad: Image without alt text --> <img src="logo.png"> <!-- Decorative images should have empty alt text --> <img src="decorative-line.png" alt="">

Form Accessibility

Make forms accessible with proper labels and instructions:

<!-- Good: Label associated with input --> <label for="username">Username:</label> <input type="text" id="username" name="username"> <!-- Bad: No label association --> <span>Username:</span> <input type="text" name="username">

ARIA Attributes

Accessible Rich Internet Applications (ARIA) attributes enhance accessibility when HTML semantics aren't sufficient:

<!-- Button with ARIA attributes --> <button aria-expanded="false" aria-controls="dropdown-menu"> Menu </button> <ul id="dropdown-menu" hidden> <li><a href="#">Item 1</a></li> <li><a href="#">Item 2</a></li> </ul>

Language Declaration

Always declare the language of your page:

<html lang="en">

Skip Navigation Links

Provide a way for keyboard users to skip repetitive content:

<a href="#main" class="skip-link">Skip to main content</a> <!-- Later in the document --> <main id="main"> <!-- Main content here --> </main>

Accessibility Testing

Test your website for accessibility using:

  • Screen Readers: NVDA, JAWS, VoiceOver
  • Browser Extensions: axe, WAVE Evaluation Tool
  • Keyboard Navigation: Try using your site with only a keyboard
  • Color Contrast Checkers: Ensure sufficient color contrast
  • Validators: HTML validators often catch accessibility issues
Remember: Accessibility is not a feature to be added at the end of development. It should be integrated into every stage of the design and development process.
