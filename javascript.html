JavaScript Basics

JavaScript is a versatile programming language primarily used for web development. It's an implementation of the ECMAScript specification and is supported by all modern browsers.

History and Evolution

JavaScript was created by Brendan Eich in 1995 while working at Netscape. It was originally named Mocha, then LiveScript, before finally being renamed JavaScript. Despite the name similarity, JavaScript is not related to Java.

Key milestones in JavaScript's evolution:

  • 1995: JavaScript created at Netscape
  • 1997: ECMAScript 1 released as first standardized version
  • 2009: ECMAScript 5 (ES5) released with significant improvements
  • 2015: ECMAScript 2015 (ES6) released with major new features
  • 2015+: Yearly ECMAScript updates with incremental improvements

Variables and Data Types

JavaScript is a dynamically typed language, meaning variables can hold values of any type without specifying the type explicitly.

Variable Declaration

// var (function-scoped, hoisted) var name = "John"; // let (block-scoped, not hoisted) let age = 30; // const (block-scoped, constant reference) const PI = 3.14159;

Data Types

JavaScript has 7 primitive data types and objects:

Type Description Example
String Textual data "Hello", 'World', `Template`
Number Numeric data (integer or floating-point) 42, 3.14, NaN, Infinity
Boolean Logical true/false values true, false
Null Intentional absence of value null
Undefined Variable declared but not assigned undefined
Symbol Unique and immutable primitive value Symbol('id')
BigInt Arbitrary precision integers 12345678901234567890n
Object Collection of key-value pairs {name: "John"}, [1, 2, 3]
Note: Use typeof operator to check a value's type, but note that typeof null returns "object" (a historical bug).

Operators

JavaScript includes various operators for performing operations on values.

Arithmetic Operators

let x = 10 + 5; // Addition: 15 let y = 10 - 5; // Subtraction: 5 let z = 10 * 5; // Multiplication: 50 let w = 10 / 5; // Division: 2 let r = 10 % 3; // Remainder: 1 let e = 2 ** 3; // Exponentiation: 8

Comparison Operators

let a = 5 == "5"; // true (loose equality) let b = 5 === "5"; // false (strict equality) let c = 5 != "5"; // false (loose inequality) let d = 5 !== "5"; // true (strict inequality) let e = 10 > 5; // true let f = 10 <= 5; // false

Logical Operators

let a = true && false; // false (AND) let b = true || false; // true (OR) let c = !true; // false (NOT) // Short-circuit evaluation let name = username || "Guest"; // Uses "Guest" if username is falsy

Control Flow

Control the execution flow of your program with conditional statements and loops.

Conditional Statements

// if-else statement if (age >= 18) { console.log("You are an adult"); } else if (age >= 13) { console.log("You are a teenager"); } else { console.log("You are a child"); } // Ternary operator let message = (age >= 18) ? "Adult" : "Minor"; // switch statement switch(dayOfWeek) { case 0: console.log("Sunday"); break; case 1: console.log("Monday"); break; default: console.log("Other day"); }

Loops

// for loop for (let i = 0; i < 5; i++) { console.log(i); } // while loop let i = 0; while (i < 5) { console.log(i); i++; } // do-while loop let j = 0; do { console.log(j); j++; } while (j < 5); // for...of loop (for iterables) let fruits = ["apple", "banana", "cherry"]; for (let fruit of fruits) { console.log(fruit); } // for...in loop (for object properties) let person = {name: "John", age: 30}; for (let key in person) { console.log(key + ": " + person[key]); }

DOM Manipulation

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the page structure as a tree of objects that can be manipulated with JavaScript.

DOM Tree Structure

The DOM represents an HTML document as a tree structure where each node is an object representing a part of the document:

// HTML structure <html> <head> <title>My Page</title> </head> <body> <h1>Welcome</h1> <p>Content here</p> </body> </html>

This would be represented as a tree with the document as the root, containing html node, which has head and body children, and so on.

Selecting Elements

JavaScript provides multiple ways to select DOM elements:

// Select by ID (returns single element) let header = document.getElementById("header"); // Select by class name (returns HTMLCollection) let items = document.getElementsByClassName("item"); // Select by tag name (returns HTMLCollection) let paragraphs = document.getElementsByTagName("p"); // Query selector (returns first matching element) let element = document.querySelector(".className"); // Query selector all (returns NodeList) let elements = document.querySelectorAll("div.item");
Note: querySelector and querySelectorAll use CSS selector syntax, making them very powerful for complex selections.

Modifying Elements

Once you've selected elements, you can modify their content, attributes, and styles.

Content Manipulation

// Change HTML content element.innerHTML = "<strong>New content</strong>"; // Change text content (safer against XSS attacks) element.textContent = "New text content"; // Get content let content = element.innerHTML; let text = element.textContent;

Attribute Manipulation

// Set attributes element.setAttribute("src", "image.jpg"); element.id = "newId"; element.className = "new-class"; // Replaces all classes // Modern class manipulation element.classList.add("new-class"); element.classList.remove("old-class"); element.classList.toggle("active"); element.classList.contains("some-class"); // Get attributes let src = element.getAttribute("src"); let id = element.id;

Style Manipulation

// Inline styles element.style.color = "blue"; element.style.fontSize = "20px"; element.style.display = "none"; // Hide element // Get computed styles let styles = window.getComputedStyle(element); let color = styles.color;
Warning: Using inline styles with the style property can lead to specificity issues. Prefer adding/removing CSS classes when possible.

Creating and Removing Elements

You can dynamically create new elements and add them to the DOM, or remove existing elements.

// Create new element let newDiv = document.createElement("div"); newDiv.textContent = "New element"; newDiv.className = "box"; // Add to DOM document.body.appendChild(newDiv); parentElement.insertBefore(newDiv, referenceElement); // Clone existing element let clonedElement = existingElement.cloneNode(true); // Remove element element.remove(); parentElement.removeChild(element);

Event Handling

JavaScript can respond to user interactions and browser events through event handlers.

Event Listeners

// Add event listener element.addEventListener("click", function(event) { console.log("Element clicked!"); }); // Remove event listener (requires named function) function handleClick(event) { console.log("Clicked"); } element.addEventListener("click", handleClick); element.removeEventListener("click", handleClick);

Common Events

Event Type Examples Description
Mouse Events click, dblclick, mousedown, mouseup, mousemove, mouseover, mouseout User interactions with mouse
Keyboard Events keydown, keyup, keypress User interactions with keyboard
Form Events submit, change, input, focus, blur Interactions with form elements
Window Events load, resize, scroll, hashchange Browser window events
Touch Events touchstart, touchmove, touchend Touch screen interactions

Event Object

When an event occurs, the browser creates an event object containing information about the event.

function handleEvent(event) { // Event properties console.log(event.type); // Event type (e.g., "click") console.log(event.target); // Element that triggered the event console.log(event.currentTarget); // Element with the event listener // Mouse event properties console.log(event.clientX, event.clientY); // Mouse coordinates console.log(event.button); // Mouse button pressed // Keyboard event properties console.log(event.key); // Key pressed console.log(event.code); // Physical key code // Event methods event.preventDefault(); // Prevent default behavior event.stopPropagation(); // Stop event bubbling }

Event Delegation

Event delegation allows you to attach a single event listener to a parent element that will fire for all descendants matching a selector, even those added dynamically.

// Instead of adding listeners to each item let items = document.querySelectorAll(".item"); items.forEach(function(item) { item.addEventListener("click", handleClick); }); // Use event delegation on parent let list = document.getElementById("myList"); list.addEventListener("click", function(event) { if (event.target.classList.contains("item")) { // Handle click on .item element console.log("Item clicked: " + event.target.textContent); } });

Functions

Functions are one of the fundamental building blocks in JavaScript. They are procedures that perform tasks or calculate values.

Function Declarations

There are several ways to define functions in JavaScript:

Function Declaration

// Function declaration (hoisted) function greet(name) { return "Hello, " + name; } // Function call let message = greet("John");

Function Expression

// Function expression (not hoisted) const greet = function(name) { return "Hello, " + name; }; // Arrow function expression const greet = (name) => { return "Hello, " + name; }; // Concise arrow function const greet = name => "Hello, " + name;

Immediately Invoked Function Expression (IIFE)

// IIFE - runs immediately after definition (function() { console.log("This runs immediately"); })(); // IIFE with parameters (function(x, y) { console.log(x + y); })(5, 10);

Function Parameters

JavaScript functions can accept parameters in flexible ways:

// Default parameters function greet(name = "Guest") { return "Hello, " + name; } // Rest parameters function sum(...numbers) { return numbers.reduce((total, num) => total + num, 0); } console.log(sum(1, 2, 3, 4)); // 10 // Destructuring parameters function displayPerson({name, age}) { console.log(`${name} is ${age} years old`); } displayPerson({name: "John", age: 30});

Return Values

Functions can return values using the return statement. If no return is specified, the function returns undefined.

// Returning a value function multiply(a, b) { return a * b; } // Returning multiple values (via array or object) function getCoordinates() { return {x: 10, y: 20}; } // Early return function isAdult(age) { if (age < 0) { return "Invalid age"; } return age >= 18; }

Scope and Closures

JavaScript has function scope, and functions can form closures by remembering their lexical environment.

Function Scope

let globalVar = "I'm global"; function example() { let localVar = "I'm local"; console.log(globalVar); // Accessible console.log(localVar); // Accessible } example(); console.log(globalVar); // Accessible console.log(localVar); // Error: localVar is not defined

Closures

function createCounter() { let count = 0; return function() { count++; return count; }; } const counter = createCounter(); console.log(counter()); // 1 console.log(counter()); // 2 console.log(counter()); // 3

Higher-Order Functions

Functions that operate on other functions, either by taking them as arguments or returning them.

// Function as argument function applyOperation(x, y, operation) { return operation(x, y); } const result = applyOperation(5, 3, (a, b) => a * b); console.log(result); // 15 // Function as return value function multiplyBy(factor) { return function(number) { return number * factor; }; } const double = multiplyBy(2); console.log(double(5)); // 10

Built-in Function Methods

JavaScript provides several methods that can be called on functions.

function greet(name, punctuation) { return `Hello, ${name}${punctuation}`; } // call() - calls function with specified this value and arguments console.log(greet.call(null, "John", "!")); // apply() - similar to call() but takes arguments as array console.log(greet.apply(null, ["John", "!"])); // bind() - creates a new function with specified this value and initial arguments const greetJohn = greet.bind(null, "John"); console.log(greetJohn("!"));

Objects & Arrays

Objects and arrays are complex data types that allow you to store and organize collections of data.

Objects

Objects are collections of key-value pairs where keys are strings (or Symbols) and values can be any data type.

Creating Objects

// Object literal (most common) let person = { name: "John", age: 30, isStudent: false, greet: function() { console.log("Hello!"); } }; // Using new Object() let car = new Object(); car.make = "Toyota"; car.model = "Camry"; // Using Object.create() let animal = Object.create(null); animal.type = "dog";

Accessing Properties

let person = {name: "John", age: 30}; // Dot notation console.log(person.name); // "John" // Bracket notation (useful for dynamic property names) let property = "age"; console.log(person[property]); // 30 // Accessing non-existent properties returns undefined console.log(person.address); // undefined

Modifying Objects

let person = {name: "John"}; // Adding properties person.age = 30; person["isStudent"] = false; // Modifying properties person.name = "Jane"; // Deleting properties delete person.age; // Checking if property exists console.log("name" in person); // true console.log(person.hasOwnProperty("name")); // true

Object Methods

let person = { firstName: "John", lastName: "Doe", // Method getFullName: function() { return `${this.firstName} ${this.lastName}`; }, // Shorthand method syntax (ES6) greet() { return "Hello!"; } }; console.log(person.getFullName()); // "John Doe"

Iterating Over Objects

let person = {name: "John", age: 30, city: "New York"}; // for...in loop for (let key in person) { console.log(key + ": " + person[key]); } // Object.keys() let keys = Object.keys(person); keys.forEach(key => { console.log(key + ": " + person[key]); }); // Object.values() (ES2017) let values = Object.values(person); console.log(values); // ["John", 30, "New York"] // Object.entries() (ES2017) let entries = Object.entries(person); entries.forEach(([key, value]) => { console.log(key + ": " + value); });

Arrays

Arrays are ordered collections of values that can be of any type.

Creating Arrays

// Array literal (most common) let fruits = ["apple", "banana", "cherry"]; // Using new Array() let numbers = new Array(1, 2, 3); // Array with mixed types let mixed = ["hello", 42, true, {name: "John"}];

Accessing Elements

let fruits = ["apple", "banana", "cherry"]; // Access by index (0-based) console.log(fruits[0]); // "apple" console.log(fruits[2]); // "cherry" // Access length console.log(fruits.length); // 3 // Access last element console.log(fruits[fruits.length - 1]); // "cherry"

Modifying Arrays

let fruits = ["apple", "banana"]; // Adding elements fruits.push("cherry"); // End: ["apple", "banana", "cherry"] fruits.unshift("orange"); // Start: ["orange", "apple", "banana", "cherry"] // Removing elements fruits.pop(); // End: ["orange", "apple", "banana"] fruits.shift(); // Start: ["apple", "banana"] // Adding/removing at any position fruits.splice(1, 0, "mango"); // Add at index 1: ["apple", "mango", "banana"] fruits.splice(1, 1); // Remove at index 1: ["apple", "banana"] // Modifying by index fruits[0] = "pear"; // ["pear", "banana"]

Array Methods

JavaScript provides many useful methods for working with arrays.

let numbers = [1, 2, 3, 4, 5]; // Iteration methods numbers.forEach(function(num) { console.log(num); }); // Transformation methods let doubled = numbers.map(num => num * 2); // [2, 4, 6, 8, 10] let evens = numbers.filter(num => num % 2 === 0); // [2, 4] let sum = numbers.reduce((total, num) => total + num, 0); // 15 // Search methods let hasThree = numbers.includes(3); // true let index = numbers.indexOf(3); // 2 // Other useful methods let reversed = numbers.reverse(); // [5, 4, 3, 2, 1] let sliced = numbers.slice(1, 3); // [2, 3] (extracts elements) let sorted = [3, 1, 4, 2].sort(); // [1, 2, 3, 4]

Multidimensional Arrays

// Creating a 2D array (matrix) let matrix = [ [1, 2, 3], [4, 5, 6], [7, 8, 9] ]; // Accessing elements console.log(matrix[0][0]); // 1 console.log(matrix[1][2]); // 6 // Iterating over 2D array for (let i = 0; i < matrix.length; i++) { for (let j = 0; j < matrix[i].length; j++) { console.log(matrix[i][j]); } }

Advanced JavaScript Topics

Once you've mastered the basics, these advanced concepts will help you write more powerful and efficient JavaScript code.

Asynchronous JavaScript

JavaScript uses an event-driven, non-blocking model for handling asynchronous operations.

Callbacks

// Traditional callback pattern function fetchData(callback) { setTimeout(() => { callback("Data received"); }, 1000); } fetchData(function(data) { console.log(data); });

Promises

// Creating a promise function fetchData() { return new Promise((resolve, reject) => { setTimeout(() => { let success = true; if (success) { resolve("Data received"); } else { reject("Error fetching data"); } }, 1000); }); } // Using a promise fetchData() .then(data => console.log(data)) .catch(error => console.error(error)) .finally(() => console.log("Operation completed"));

Async/Await

// Async function (returns a promise) async function getData() { try { let data = await fetchData(); console.log(data); return data; } catch (error) { console.error(error); throw error; } } getData();

ES6+ Features

Modern JavaScript has introduced many new features that improve code readability and functionality.

Destructuring

// Array destructuring let [a, b, c] = [1, 2, 3]; console.log(a, b, c); // 1, 2, 3 // Object destructuring let {name, age} = {name: "John", age: 30}; console.log(name, age); // "John", 30 // Function parameter destructuring function greet({name, age}) { console.log(`${name} is ${age} years old`); }

Template Literals

let name = "John"; let age = 30; // String interpolation let message = `${name} is ${age} years old`; // Multi-line strings let multiline = ` This is a multi-line string `;

Spread and Rest Operators

// Spread operator (arrays) let arr1 = [1, 2, 3]; let arr2 = [4, 5, 6]; let combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6] // Spread operator (objects) let obj1 = {a: 1, b: 2}; let obj2 = {c: 3, d: 4}; let merged = {...obj1, ...obj2}; // {a: 1, b: 2, c: 3, d: 4} // Rest parameters function sum(...numbers) { return numbers.reduce((total, num) => total + num, 0); }

Modules

ES6 introduced a native module system for JavaScript.

// math.js (exporting) export const PI = 3.14159; export function square(x) { return x * x; } export default function multiply(x, y) { return x * y; } // app.js (importing) import multiply, { PI, square } from './math.js'; console.log(PI); // 3.14159 console.log(square(5)); // 25 console.log(multiply(4, 7)); // 28

Error Handling

Proper error handling is essential for robust applications.

// Try-catch-finally try { // Code that might throw an error let result = riskyOperation(); console.log(result); } catch (error) { // Handle the error console.error("An error occurred:", error.message); } finally { // Code that runs regardless of success or failure console.log("Operation attempted"); } // Throwing custom errors function validateInput(input) { if (!input) { throw new Error("Input cannot be empty"); } if (input.length < 5) { throw new Error("Input must be at least 5 characters"); } }
